	Installation

	-----------------

	If not done, upgrade to the latest version of Postnuke (0.726-3 as writing
   	this on 21th Jun 2004)



	Install the XTE (Xantha Theme Engine) incl. pnRender (you can download it from http://www.post-nuke.net or other PN related sites)



    Install the latest Mailer module (pnuserapi.php min. version 1.14!!) and activate it. This module is part of the coming Postnuke .8 and Postnuke .750 Gold and will work in .726-3 too.


        
	Extract the files to your Postnuke directory.



	Load, initialize and activate the module in the admin section.



	Create some contact names/topics with respective email adresses	in the Formicula admin module.



	Change some settings in the contact configuration (i.e. activate/deactivate some extra fields or sending confirmation email to the user).



	Add a link to your main menu with: {formicula}


   	(using [formicula] will not work)


   	or somewhere else:  
	
	index.php?module=formicula&func=main 



	If neccesary, change the language files to your needs.



	If you do not like the page layout: go to /pntemplates and change the files to fit your needs. This is Smarty!


	
	Templates

	----------------

	The templates for the forms are named as follows:


	The form with the input fields:  #_userform.html

	The page with the confirmation after submitting the data:  #_userconfirm.html

	The page with error messages after submitting data:  #_usererror.html

	The user's confirmation mail in text format:  #_usermail.txt

	The user's confirmation mail in html format:  #_usermail.html

	The admin mail in text format:  #_adminmail.txt

	The admin mail in html format:  #_adminmail.html

	This is the email that the admin gets after the user sends the mail.

	Note: the email format can be chosen by using the dropdown that we have in 
	our sample 0_userform.html


				
				    <select name='userformat'>

				    <option value='html'><!--[ pnml name='_FOR_HTMLMAIL' ]--></option>

				    <option value='plain'><!--[ pnml name='_FOR_TEXTMAIL' ]--></option>

				    </select>


	Or if you want to force the type of email, you can use a hidden field in the form.


				
				    <input type='hidden' name='userformat' value='html'>


	The '#' at the beginning of the template designates the number of form

	

	You can call the specific form with a parameter
	 &form=# in the url. 
	A link from the menu to call form #1 would look like:


	
				    index.php?module=formicula&func=main&form=1 


	

	In the distribution package you will find 2 sample forms 0 and 1, 
	0 is a normal contact form and form #1 is for online a job application.
	If form is not mentioned in the url, the default is 0. 
	So if you want to use Formicula for only one contact form, 
	the link in the menu can be {formicula}


	

	Custom Fields

	----------------------

	You can add as many custom fields to your form as you want. 
	This makes it easy to create e.g. an online job application form if needed.
	This makes it necessary to send certain information from the form page to the module.


	

	These are:

	
	numFields (hidden field) number of custom fields used

	custom#name (hidden field) name of the custom field

	custom#mandatory (hidden field) set to 1 if this is a mandatory field. 
	Formicula checks this and shows an errormessage when this field is not filled

	custom#data (any kind of input field) the data with #=number.


	Example:


	
	  <tr width='100%'>

	    <td width='50%' align='right'>

	      <!--[ pnml name='_ONLINEAPPLYAS' html=1 ]-->&nbsp;<span style="color: red;">*</span>

	    </td>

	    <td width='50%' align='left'>

	      <input type='hidden' name='custom0name' value='<!--[ pnml name='_ONLINEAPPLYAS' ]-->'>

	      <input type='hidden' name='custom0mandatory' value=1>

	      <input type='text' name='custom0data' size='40' maxlength='80' value=''>

	    </td>

	  </tr>



	  <tr width='100%'>

	    <td width='50%' align='right'>

	      <!--[ pnml name='_ONLINESTREET' html=1 ]-->&nbsp;<span style="color: red;">*</span>

	    </td>

	    <td width='50%' align='left'>

	      <input type='hidden' name='custom1name' value='<!--[ pnml name='_ONLINESTREET' ]-->'>

	      <input type='hidden' name='custom1mandatory' value=1>

	      <input type='text' name='custom1data' size='40' maxlength='40' value=''>

	    </td>

	  </tr>


	
	
	This will create two custom fields 0 and 1 which are both mandatory. 
	In this case you also need:


	
	    <input type='hidden' name='numFields' value=2>



	Please note that the customfields have to start at 0!
	

    

	

	Permissions

	------------------

	To use a form you need the right to COMMENT.

	
	
Unregistered   |   formicula::  |  .*   |  comment


	all unregistered users are allowed to use all forms and write to all contacts

	
	
Unregistered   |   formicula::  |  0::  |  comment


	all unregistered users are allowed to use form 0 with all existing contacts.

	

Users   |   formicula::   |  1:(2|3):  |  comment


    all members of the group users are allowed to use form 1 and write to contact 2 and 3.
